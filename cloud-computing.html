Detailed Syllabus

What You Will Study in This Diploma

Below is the detailed breakdown of the modules so students can clearly understand what they will learn and why each module matters for their career.

Module 1

Hardware Fundamentals

Hardware is the first step in understanding IT. If a student wants to work in cloud, networking, support, or system administration, they should first know how a computer system actually works from inside.

Why Learn Hardware?

  • Builds technical confidence from the ground level
  • Helps students understand how systems are assembled and maintained
  • Useful for troubleshooting, support, and infrastructure roles
  • Creates a strong base before moving into server and cloud environments

Topics Covered

  • Computer components and architecture
  • Motherboard, CPU, RAM, HDD, SSD, SMPS
  • Input/output devices and peripherals
  • BIOS/UEFI basics
  • Assembling and disassembling systems
  • Basic troubleshooting and maintenance
Module 2

Client Operating System

Students must understand how desktop and laptop operating systems are used in daily IT work. This module helps learners become comfortable with system setup, troubleshooting, and user-side technical support.

Why Learn Client OS?

  • Useful for IT support and desktop administration jobs
  • Improves understanding of software, drivers, users, and permissions
  • Builds a practical base before learning servers and Linux

Topics Covered

  • Windows installation and formatting
  • User accounts and system settings
  • Drivers, updates, and software installation
  • Control Panel and basic administration tools
  • Disk management and file systems
  • Common troubleshooting techniques
Module 3

Networking Fundamentals (N+ Level)

Networking is one of the most important modules because cloud systems, servers, and infrastructure all depend on networks. Students will learn the logic behind how devices communicate in real environments.

Why Learn Networking?

  • Required for cloud, Linux, server, and support roles
  • Helps students understand connectivity and troubleshooting
  • Builds a strong technical mindset for IT infrastructure

Topics Covered

  • What is networking and how networks work
  • LAN, WAN, MAN and topologies
  • OSI Model and TCP/IP Model
  • IP addressing and subnetting basics
  • Switches, routers, cables, and wireless concepts
  • Ports, protocols, DNS, DHCP, and troubleshooting basics
Module 4

CCNA 200-301

This module gives students exposure to professional Cisco networking concepts. It helps learners move beyond networking basics and understand the enterprise-level side of networking and connectivity.

Why Learn CCNA?

  • Widely recognized networking knowledge path
  • Helpful for networking, support, and infrastructure roles
  • Improves confidence in network design and troubleshooting

Topics Covered

  • Network fundamentals and Cisco basics
  • Switching concepts and VLAN understanding
  • Routing concepts and IP connectivity
  • Network access and IP services
  • Security fundamentals
  • Automation and programmability overview
Module 5

Server Operating System

Servers are the backbone of most company environments. This module introduces students to enterprise server administration and how organizations manage users, systems, and services in centralized environments.

Why Learn Server OS?

  • Important for system administration and infrastructure careers
  • Helps students understand centralized IT environments
  • Creates a direct bridge toward cloud administration concepts

Topics Covered

  • Introduction to Windows Server
  • Active Directory basics
  • DNS and DHCP roles
  • Users, groups, and permissions
  • Group Policy concepts
  • Server roles and administrative tools
Module 6

Python Basics for IT Students

Python is added so students do not remain limited to only support-level knowledge. Even basic coding improves logic, automation thinking, and future career opportunities in cloud, DevOps, and data roles.

Why Learn Python?

  • Builds coding confidence for beginners
  • Useful for automation and scripting in modern IT roles
  • Supports future learning in cloud, DevOps, and analytics

Topics Covered

  • Variables, data types, and operators
  • Conditions and loops
  • Functions and basic logic building
  • Lists, tuples, and dictionaries
  • Basic file handling
  • Simple scripting practice
Module 7

Linux Operating System

Linux is one of the most valuable skills for students who want higher technical roles and better career opportunities. Many servers, cloud platforms, and production environments depend on Linux.

Why Learn Linux?

  • Highly valuable for jobs in cloud, DevOps, servers, and hosting
  • Improves command-line confidence and system understanding
  • Often associated with better technical growth and salary potential

Topics Covered

  • Linux installation and introduction
  • Basic commands and file navigation
  • Users, groups, and permissions
  • Package management and services
  • Process management and basic networking
  • Directory structure and administration basics
Module 8

Cloud Computing with AWS + Azure Introduction

This is the core module where students connect all previous learning and understand how cloud platforms are used in modern companies. Students learn what cloud is, why companies use it, and how services are deployed and managed.

Why Learn Cloud Computing?

  • Cloud is one of the fastest-growing areas in IT
  • Companies are moving workloads to platforms like AWS and Azure
  • Cloud knowledge increases career opportunities in modern infrastructure roles

Topics Covered

  • What is cloud computing and why it matters
  • Cloud service models: IaaS, PaaS, SaaS
  • Deployment models: Public, Private, Hybrid
  • AWS basics: IAM, EC2, S3, VPC, Security Groups
  • Storage, networking, and identity fundamentals
  • Azure overview and cloud platform comparison basics
